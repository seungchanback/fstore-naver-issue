# 소개
---

2022년 4월까지 S3(cdata_day2) 에 적재된 데이터에 이슈가 있는 것을 확인하였습니다.
분석가분들은 개별적으로 전처리를 진행하시거나, 이슈를 기록해주셨습니다.
개별적으로 작성해주신 내용을 취합해서 S3 의 전체 탐색을 위해 코드를 작성하였습니다.


해당 레포지토리는 아래의 목적을 위해 개발되었습니다.
- S3 의 파일의 csv 파일의 전체 탐색
- 표준출력으로 문제가 되는 filepath 와 관련 내용 출력
- 아래의 이슈에 대한 문제점 파악
  1. 키워드가 파일이름과 일치하지 않는 파일 확인
  2. 파일 내 행이 다른 행과 형태가 다른지 확인
  3. rank 값이 200 이상인 데이터가 포함되어 있는지 확인
  4. keyword 에 Null 값이 포함되어 있는지 확인


# 실행 순서
---

1. `config.ini` 에  AWS 설정내용을 입력합니다.
2. `main.py`의 CHECK_ITEM 은 cdata_day2 속에 검사할 폴더 목록입니다. 검사할 폴더 목록으로 수정합니다.
3. 'python3 main.py` 를 실행하면 TEMP 폴더가 생성되고 S3 를 다운로드합니다.
4. 검사를 하고 표준출력으로 결과를 출력합니다. 검사가 완료되면 TEMP 폴더는 삭제됩니다.
